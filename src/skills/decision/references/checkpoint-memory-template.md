# Checkpoint Memory Template

Use this when later turns could otherwise resume from the wrong node, route, or blocker.

## Recommended structure

### 1. Current route

- current route label
- current stage / anchor
- short route judgment

### 2. Current active node

- active branch / run / paper node / report pair
- why this is the authoritative resume point now

### 3. Node history

- predecessor node(s)
- superseded node(s) or route(s)
- why the current node beat or replaced them

### 4. Strongest retained result or blocker

- strongest still-valid result
- or the dominant current blocker
- why it matters

### 5. Do-not-reopen by default

- experiments, routes, or closure paths that should stay closed unless new evidence appears
- any explicit do-not-repeat rule

### 6. Next resume step

- the single best next step
- why it dominates alternatives

### 7. First-read files

- `status.md`
- `SUMMARY.md`
- current `PLAN.md` / `CHECKLIST.md`
- any route-critical report, manifest, or handoff file

### 8. Reopen condition

- exact condition that would justify reopening an older node or switching away from the current one
