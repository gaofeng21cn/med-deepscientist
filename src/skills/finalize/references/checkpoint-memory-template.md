# Checkpoint Memory Template

Use this when the quest is pausing, entering continue-later, or otherwise needs one compact memory card that says exactly where later turns should resume.

## Recommended structure

### 1. Current route

- current route label
- current stage / anchor
- short closure or continue-later judgment

### 2. Current active node

- active paper line / run node / report-decision pair
- why this is the authoritative checkpoint now

### 3. Node history

- predecessor node(s)
- superseded completion, experiment, or writing paths
- why the current checkpoint replaced them

### 4. Strongest retained result or blocker

- strongest still-valid result
- or the dominant blocker
- why it matters

### 5. Do-not-reopen by default

- experiments, branches, or closure paths that should stay closed unless new evidence appears
- any explicit do-not-repeat rule

### 6. Next resume step

- the single best next step
- why it dominates alternatives

### 7. First-read files

- `status.md`
- `SUMMARY.md`
- active resume / handoff packet
- current manifest, report, or decision file that defines the live checkpoint

### 8. Reopen condition

- exact condition that would justify reopening an older node or switching away from the current checkpoint
